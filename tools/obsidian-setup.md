# Obsidian：让你的知识库可以快速检索

> 内容来源：自编

## Obsidian 是什么

Obsidian 是一个**本地 Markdown 编辑器 + 知识管理工具**。它直接读取你电脑上的文件夹，提供快速搜索、预览、双向链接等功能。

核心特点：
- **本地优先** — 所有数据就是你硬盘上的 `.md` 文件，不存云端
- **快速检索** — 模糊搜索秒开笔记，全文搜索定位内容
- **双向链接** — 笔记之间可以互相引用，形成知识网络
- **插件生态** — 社区提供上千款插件，按需扩展

## 对我现有工作流的帮助

### 现有工作流（继续保留）

```
在本地 GitHub 仓库中工作
  → Claude Code 辅助整理、总结
  → git commit + push 到远程仓库
  → 形成知识库
```

### 痛点

**笔记用时找不到** — 需要手动翻目录或用命令行搜索，没有即时调出的体验。

### Obsidian 的角色

Obsidian **不替代**现有工作流，而是叠加在上面：

| 环节 | 工具 |
|------|------|
| **生产**（写笔记、整理） | Claude Code + Git |
| **消费**（查笔记、读笔记） | Obsidian |

只需要用 Obsidian 打开同一个文件夹，就能同时拥有两套工具的优势，零迁移成本。

> 进阶：如果想让这个仓库不只是"看笔记"而是"持续编译知识"，参见 [LLM Wiki 知识库方法论](../workflow/llm-wiki-methodology.md)。

## 安装

### macOS

**方法一：官网下载（推荐）**

1. 打开 [https://obsidian.md](https://obsidian.md)，点击 Download，选择 macOS 版本
2. 下载 `.dmg` 文件，双击打开
3. 将 Obsidian 拖入「应用程序」文件夹
4. 启动台找到 Obsidian，点击打开

**方法二：App Store**

直接在 Mac App Store 搜索「Obsidian」，点击获取安装。

### 验证安装

打开 Obsidian，能看到「创建新库」或「打开已有库」的界面即表示安装成功。

## 使用：打开现有知识库

### 第一步：用 Obsidian 打开你的 GitHub 仓库

1. 打开 Obsidian
2. 选择「打开已有库」（Open folder as vault）
3. 选择你的知识库目录，例如：`~/Documents/Github/okj-claude-ai`
4. 点击「打开」

> Obsidian 会在仓库根目录创建 `.obsidian/` 隐藏目录存放配置。已在 `.gitignore` 中排除，不会影响 Git 仓库。

### 第二步：掌握三个核心快捷键

这三个快捷键解决你「笔记用时找不到」的问题：

| 快捷键 | 功能 | 使用场景 |
|--------|------|---------|
| `Cmd + O` | **快速切换** — 输入文件名模糊搜索，秒开笔记 | 知道笔记大概叫什么名字 |
| `Cmd + Shift + F` | **全库搜索** — 按内容关键词搜索所有笔记 | 只记得某个关键词 |
| `Cmd + P` | **命令面板** — 输入任何操作名称执行 | 不记得快捷键时用这个 |

### 第三步：日常使用方式

**查笔记时：**

1. 打开 Obsidian
2. `Cmd + O` 输入关键词（如 `gstack`、`hook`、`channel`）
3. 回车打开笔记，阅读内容

**写笔记时（不变）：**

1. 终端进入仓库目录
2. 用 Claude Code 辅助整理、生成文档
3. `git commit` + `git push`
4. Obsidian 会自动感知文件变化，无需刷新

## 推荐配置

### 切换中文界面

1. `Cmd + ,` 打开设置
2. 左侧选择「About」
3. Language 选择「简体中文」
4. 重启 Obsidian

### 推荐安装的插件（只装两个）

打开设置 → 第三方插件 → 关闭安全模式 → 浏览

| 插件 | 用途 |
|------|------|
| **Minimal Theme** | 极简主题，界面清爽 |
| **Dataview** | 笔记多了以后可以自动汇总分类 |

> 不要贪多装插件。你的笔记生产靠 Claude Code，不需要模板、日历等插件。插件装多了反而拖慢启动速度，分散注意力。

## 进阶：双向链接（可选）

如果你想让笔记之间建立关联，可以在 Markdown 中使用 `[[笔记名]]` 语法：

```markdown
安装 gstack 需要先配置好开发环境，参见 [[installation]]
```

点击链接可以直接跳转到对应笔记。但这不是必须的 — **快速搜索已经能解决 90% 的检索需求**。

## 常见问题

**Q：Obsidian 会修改我的 Markdown 文件吗？**

不会。Obsidian 只是读取和显示你的文件。除非你在 Obsidian 里编辑，否则文件内容不会变化。

**Q：用了 Obsidian 还需要 Claude Code 吗？**

当然需要。Obsidian 负责「看」，Claude Code 负责「写」。两个工具各司其职。

**Q：`.obsidian/` 目录会不会影响 Git？**

不会。已经在 `.gitignore` 中排除了这个目录。

**Q：需要付费吗？**

核心功能完全免费。付费的是官方同步服务（Obsidian Sync），但你已经有 Git 同步，不需要购买。
