# 新增学习文档 SOP

## 执行步骤

1. **确认分类**：根据内容类型确定文档放入哪个目录：

   | 目录 | 适合内容 |
   |------|---------|
   | `setup/` | 安装、初始配置、环境准备 |
   | `usage/` | 命令、快捷键、日常操作 |
   | `channels/` | Telegram / Discord 远程控制 |
   | `plugins/` | 插件安装与使用 |
   | `tools/` | 具体工具（gstack 等） |
   | `quickstart/` | 综合快速入门指南 |
   | `workflow/` | 工作流、方法论、最佳实践 |

   不属于现有分类时，先建新目录再写文档。

2. **确认文件名**：用小写英文 + 连字符命名，体现主题核心，如 `mcp-server-setup.md`。

3. **按模板写文档**：

```markdown
# [主题标题]

## 概述

[一句话说明这个功能/工具是什么，解决什么问题]

## 前提条件

[需要提前准备什么]

## 配置 / 安装步骤

[具体操作，写实际可执行的命令]

## 使用方法

[典型使用场景和示例]

## 注意事项

> [踩坑点或需要特别注意的地方]
```

4. **更新 README.md 索引**：在对应分类下添加链接。

## 注意事项

- **不要编造命令**：不确定的命令先查文档或问用户，不要写可能不工作的示例
- **来源要标注**：第三方文章整理的内容在文档开头注明原始来源
- **保持精简**：学习文档以够用为原则，不要堆砌所有细节
