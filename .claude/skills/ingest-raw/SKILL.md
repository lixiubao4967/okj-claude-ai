# 原始素材 Ingest SOP

## 输入

- 一个或多个 `raw/` 下的文件路径（用户指定）

## 前置检查（不可跳过）

执行任何 ingest 动作之前，先检查 `raw/` 目标文件是否已被 git 跟踪：

```bash
git ls-files --error-unmatch raw/<目标文件>
```

- 已跟踪且无未提交修改：可以继续
- 未跟踪 / 有未提交修改：**停下来**，提示用户先 commit raw 文件
  - 原因：第 6 步会删除该文件，原文只能依赖 git 历史溯源，未 commit 会导致原始素材永久丢失

## 执行步骤

### 1. 读取并理解原始素材

- 读完整篇内容，识别核心观点、关键概念、可执行步骤
- 判断内容来源类型，套用 CLAUDE.md 的来源标注：
  - `自编`：用户原创笔记或对话整理
  - `官方`：Anthropic 官方文档/blog
  - `第三方文章解读`：公众号、博客、个人文章

### 2. 重叠检测（合并优先策略）

用 Grep/Glob 检索现有文档（按分类目录扫描标题和章节标题）：

- **默认行为：合并** —— 只要存在主题相关的现有文档，就把素材的相关观点追加进去
- **拆分合并**：素材包含多个独立主题 → 分别合并到不同文档，单篇 raw 可以触发多文件改动
- **新建例外**：当且仅当现有目录中确实没有任何相关文档时才新建。新建必须在汇报中给出"为何不能合并"的明确理由

### 3. 分类归属决策（单一归属 + 交叉链接）

参考 CLAUDE.md 的目录结构（`setup/ usage/ channels/ plugins/ tools/ quickstart/ workflow/ internals/`）：

1. 跨分类概念在 8 个分类中选**最贴切的一个**作为主分类，文档只放这里
2. 在其他相关分类的相关文档中，用 markdown 链接 `[概念名](../主分类/文档.md)` 交叉引用
3. **不允许**在多个分类目录复制相同内容（避免后续维护双份）

### 4. 写入或修改文档

- 新建：套用 doc-standards.md 的格式规范和 add-learning-doc 的模板
- 修改：在原文档合适位置追加章节，标注来源
- 文档开头（或追加章节开头）务必标注内容来源，**只保留标题和作者，不写链接**：

  ```markdown
  > 来源：第三方文章解读 · 《文章标题》 · 作者：xxx · ingest 自 raw/2026-05-04-xxx.md
  ```

  - 来源类型：`自编` / `官方` / `第三方文章解读`（来自 CLAUDE.md）
  - 不保留 URL 的原因：公众号链接易失效，原文已通过 git 历史的 raw/ 文件保存
  - 作者未知时写 `作者：未知`，不留空

### 5. 更新 README.md 索引

- 新建文档时：在对应分类章节添加链接
- 扩充已有文档时：通常不动 README，除非新增了独立小节

### 6. 删除 raw/ 原始文件

ingest 完成后，删除 raw/ 下被处理的原始文件：

```bash
rm raw/<目标文件>
```

- 原始内容通过 git 历史保留（前置检查已确保文件已 commit）
- 用户后续可通过 `git log raw/` 查看 ingest 历史
- 通过 `git show <commit>:raw/<文件>` 取回任意时点的原始素材

### 7. 输出汇报

ingest 结束后，向用户汇报：

- **新增文件**：路径列表（每条说明为何不能合并）
- **修改文件**：路径 + 修改章节
- **跳过的内容**：哪些观点判断重叠太低或太碎，没有合并
- **删除的 raw 文件**：路径列表
- **未决问题**：分类不确定、需要用户拍板的概念归属

## 注意事项

- **raw/ 进 git**：本仓库 raw/ 纳入版本控制，`.gitignore` 不应排除 raw/
- **先 commit 再 ingest**：前置检查不可跳过，否则原始素材永久丢失
- **不擅自删除现有内容**：合并时只追加或局部改写，不整段重写已有文档
- **不编造命令**：原始素材里出现的命令如果不能验证，标注 `> 来源未验证` 而不是直接收录
- **保持精简**：原始素材常啰嗦，提炼时只保留对本仓库读者有价值的部分
