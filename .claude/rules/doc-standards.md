# 文档写作规范

## 结构要求

- 每篇文档以 `# 标题` 开头，清晰说明主题
- 安装类文档按 **前提条件 → 安装步骤 → 验证** 组织
- 指南类文档按 **概念 → 配置 → 使用示例** 组织
- 用 `---` 分隔不同主题段落

## 格式规范

- CLI 命令用 ```bash 代码块
- 配置文件内容用对应语言代码块（json、yaml 等）
- 命令名、文件名、参数用 `行内代码`
- 关键结论或注意事项用 **粗体** 或 > blockquote 标注

## 内容规范

- 不用占位符，写实际可执行的命令
- 踩坑经验用 > blockquote 标注，方便快速定位
- 引用外部资料注明来源

## Commit 规范

- 统一使用 `docs:` 前缀
- 格式：`docs: 动词 + 简短描述`，例如 `docs: 添加 gstack 安装指南`
